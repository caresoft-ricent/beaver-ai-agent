## 一、系统登录与初始化

### 1.1 首次使用

首次访问系统时，需要初始化管理员账号：

1. 打开浏览器，访问 `http://localhost:3000`
2. 系统自动跳转到登录页面
3. 点击「初始化管理员」按钮
4. 设置管理员用户名和密码
5. 初始化完成后自动跳转到登录页面

### 1.2 登录

1. 输入用户名和密码
2. 点击「登录」按钮
3. 登录成功后进入仪表盘

### 1.3 修改密码

登录后，在页面右上角可以进行密码修改。

## 三、租户管理

**路径：** 左侧菜单 → 租户管理

租户是系统的基础配置单元。每个客户对应一个租户，所有配置数据（连接器、大模型、技能等）均按租户隔离。

### 3.1 租户列表

- 支持按名称/编码搜索
- 支持分页浏览
- 列表展示：租户名称、编码、状态、创建时间

### 3.2 新建租户

1. 点击「新建」按钮
2. 填写租户信息：
   - **租户名称**：客户的展示名称
   - **租户编码**：唯一标识符，建议使用英文
   - **状态**：启用/禁用
   - **配置**：JSON 格式的扩展配置（可选）
3. 点击「确定」保存

### 3.3 编辑/删除

- 在表格操作列，点击「编辑」修改租户信息
- 点击「删除」可删除租户（会级联影响关联数据，请谨慎操作）

## 四、连接器配置

**路径：** 左侧菜单 → 连接器管理

连接器用于对接外部系统（如河狸云 API），是工具调用的桥梁。

### 4.1 新建连接器

1. 点击「新建」按钮
2. 填写连接器信息：
   - **名称**：连接器展示名称
   - **编码**：唯一标识符
   - **所属租户**：选择关联的租户
   - **基础 URL**：外部系统的 API 基础地址（如 `https://api.example.com`）
   - **认证类型**：Header / Bearer / API Key
   - **认证信息**：Header 名称、Token 值等
   - **Mock 模式**：开启后不实际调用外部 API，使用 Mock 数据返回

### 4.2 连接测试

1. 在连接器列表，点击操作列的「测试」按钮
2. 系统会尝试调用连接器的健康检查路径
3. 返回测试结果（成功/失败及详细信息）

> 提示：Mock 模式下测试始终返回成功，适合无 API 环境的演示。

### 4.3 Mock 模式说明

启用 Mock 模式后，所有通过该连接器的 API 调用都会返回预设 Mock 数据:
- 无需外部 API 可用即可完整体验系统功能
- 适合开发环境和演示场景
- 可随时关闭 Mock 切换到真实 API 调用

## 五、大模型配置

**路径：** 左侧菜单 → 大模型配置

配置系统使用的 LLM 大语言模型，用于意图识别兜底、回答生成等场景。

### 5.1 新建大模型配置

1. 点击「新建」按钮
2. 填写配置信息：
   - **配置名称**：自定义名称
   - **所属租户**：关联的租户
   - **提供商**：选择 LLM 提供商（OpenAI / MiniMax / LM Studio 等）
   - **模型名称**：具体模型标识（如 `gpt-4`、`abab6-chat`）
   - **API 地址**：模型接口地址
   - **API Key**：认证密钥
   - **用途**：选择使用场景
     - `intent` — 意图识别
     - `reply` — 回答生成
     - `entity` — 实体抽取
     - `general` — 通用

### 5.2 模型测试

1. 在配置列表，点击「测试」按钮
2. 系统向该模型发送测试请求
3. 返回模型响应结果，确认配置正确

### 5.3 支持的提供商

| 提供商 | 说明 |
|--------|------|
| OpenAI | 标准 OpenAI API 兼容接口 |
| MiniMax | MiniMax 模型（推荐用于中文意图识别） |
| LM Studio | 本地部署模型（开发测试用） |

## 六、业务本体管理

**路径：** 左侧菜单 → 业务本体

业务本体是对外部系统业务对象的抽象描述（如"产线"、"人员"），包含属性定义和操作（API）定义。

### 6.1 核心概念

```
实体 (Entity)
 ├── 属性 (Property)      → 描述实体的字段（名称、类型、是否输入/输出）
 ├── 操作 (Action)         → 实体上可执行的 API 操作
 │    └── 参数 (Parameter) → 操作的输入参数
 └── 关系 (Relation)       → 实体间的关联关系
```

### 6.2 创建实体

1. 点击「新建实体」
2. 填写实体信息：
   - **实体编码**：唯一标识，如 `production_line`
   - **实体名称**：中文名称，如 "产线"
   - **所属租户**：选择租户
   - **连接器**：选择关联的连接器
   - **描述**：业务说明

### 6.3 管理属性

1. 在实体列表，点击实体展开详情
2. 点击「管理属性」打开弹窗
3. 添加属性时需指定：
   - **属性名称**：如 `line_code`
   - **类型**：string / number / date / boolean 等
   - **是否输入参数**：调用 API 时是否作为查询条件
   - **是否输出字段**：API 返回结果中是否包含

### 6.4 管理操作

1. 点击「管理操作」打开弹窗
2. 添加操作时需指定：
   - **操作编码**：如 `query_progress`
   - **操作名称**：如 "查询进度"
   - **HTTP 方法**：GET / POST / PUT / DELETE
   - **API 路径**：相对于连接器 Base URL 的路径
3. 为操作添加参数：
   - 参数名、参数类型、是否必填

### 6.5 发布实体

实体配置完成后，点击「发布」使其在技能配置中可用。

> 注意：修改已发布实体的结构后需要重新发布。

## 七、技能/意图管理

**路径：** 左侧菜单 → 技能管理

技能定义了系统能处理的用户意图，以及对应的处理方式。

### 7.1 技能类型

系统支持两种技能模式：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **简单模式** | 按工具链顺序执行操作 | 单步查询、简单操作 |
| **流程编排模式** | 使用可视化有向图定义执行流程 | 多步骤、有分支的复杂流程 |

### 7.2 创建技能

1. 点击「新建」按钮
2. 填写技能信息：
   - **技能编码**：唯一标识，如 `QUERY_PROGRESS`
   - **技能名称**：中文名称，如 "查询产线进度"
   - **所属租户**：选择租户
   - **匹配关键词**：多个关键词用逗号分隔，如"进度,交货,产线"
   - **正则表达式**：可选，用于精确匹配（如 `.*进度.*`）
   - **回答模板**：可选，固定回答模板
   - **优先级**：数字越大优先级越高
   - **描述**：技能说明

### 7.3 配置工具链

简单模式下，为技能关联业务本体的操作：

1. 在技能详情中，点击「管理工具」
2. 添加工具关联：
   - **本体**：选择业务实体
   - **操作**：选择该实体下的操作
   - **工具模式**：`api`（接口调用直接包装）或 `entity`（本体引用）
3. 多个工具按顺序依次执行

### 7.4 切换到流程编排

当需要更复杂的逻辑（条件分支、并行执行等）时：

1. 在技能列表中，找到目标技能
2. 点击操作列的「流程编排」按钮
3. 系统跳转到全屏流程编排页面
4. 详细操作见 [第八章 流程编排器](#八流程编排器)

### 7.5 发布技能

技能配置完成后，点击「发布」使其在对话中生效。

> 注意：只有已发布的技能才会参与意图匹配。

## 八、流程编排器

**路径：** 技能管理 → 点击目标技能的「流程编排」 → 全屏编辑页面

流程编排器是系统的核心可视化功能，允许通过拖拽方式定义复杂的自动化流程。

### 8.1 界面布局

```
┌─────────────────────────────────────────────────────────────┐
│ [←返回]  技能名称 / 技能编码           [更多▾]  [保存]     │
├──────────┬──────────────────────────────────┬───────────────┤
│          │                                  │               │
│  左侧    │          画布区域                 │   右侧        │
│  节点    │     (拖拽/连线/编辑)              │   属性面板    │
│  面板    │                                  │               │
│          │                                  │               │
├──────────┴──────────────────────────────────┴───────────────┤
│ 缩放控制 | 小地图                                            │
└─────────────────────────────────────────────────────────────┘
```

### 8.2 节点类型

| 节点 | 颜色 | 用途 | 基本配置 |
|------|------|------|---------|
| 🔵 工具调用 | 蓝色 | 调用本体操作 | 选择本体 → 选择操作 |
| 🟡 条件判断 | 琥珀色 | 分支路由 | 判断字段 + 分支条件 + 默认分支 |
| 🟣 并行执行 | 紫色 | 并行多任务 | 选择要并行执行的节点 |
| 🟢 用户确认 | 绿色 | 暂停确认 | 确认标题 + 消息 + 拒绝跳转 |
| 🩷 AI 生成 | 粉色 | LLM 生成 | 提示词 + 最大 Token |
| 🔹 文本回复 | 青色 | 直接回复 | 回复文本模板 |

### 8.3 添加节点

**方式一：拖拽添加**
1. 从左侧面板找到目标节点类型
2. 按住鼠标拖动到画布中的目标位置
3. 松开鼠标完成添加

**方式二：点击添加**
1. 在左侧面板直接点击节点类型
2. 节点自动添加到画布中

**方式三：右键菜单**
- 在画布空白区域操作

### 8.4 连接节点

1. 将鼠标移到节点右侧的输出端口（空心圆）
2. 按住鼠标拖动到目标节点左侧的输入端口（实心圆）
3. 松开鼠标完成连线
4. 连线表示流程的执行方向

> 条件判断节点有额外的底部端口，用于默认分支连线。

### 8.5 编辑节点

1. 点击画布中的节点
2. 右侧属性面板显示该节点的配置
3. 修改节点名称、参数等配置
4. 修改实时保存

### 8.6 节点配置详解

#### 工具调用 (tool_call)

| 字段 | 说明 |
|------|------|
| 节点名称 | 显示在节点卡片上 |
| 本体 | 选择调用的业务实体 |
| 操作 | 选择该实体下的操作（选择本体后自动加载） |

#### 条件判断 (condition)

| 字段 | 说明 |
|------|------|
| 判断字段 | 引用前序节点结果，如 `tool_results.node_1.data.status` |
| 条件分支 | 添加多个条件：操作符(=, !=, 包含, >, <) + 值 + 跳转节点 |
| 默认分支 | 所有条件不匹配时的跳转节点 |

#### 并行执行 (parallel)

| 字段 | 说明 |
|------|------|
| 并行节点 | 选择要同时执行的多个节点 |

#### 用户确认 (confirm)

| 字段 | 说明 |
|------|------|
| 确认标题 | 弹窗标题 |
| 确认消息 | 展示给用户的消息，支持引用前序结果 |
| 拒绝后跳转 | 用户点击取消后的跳转节点 |

#### AI 生成 (llm_call)

| 字段 | 说明 |
|------|------|
| 提示词 | 发送给大模型的提示文本，支持变量引用 |
| 最大 Token | 生成文本的长度限制（64~8192） |

#### 文本回复 (reply)

| 字段 | 说明 |
|------|------|
| 回复文本 | 直接输出的模板文本，支持变量替换 |

### 8.7 快捷操作

**键盘快捷键：**

| 快捷键 | 功能 |
|--------|------|
| `Ctrl/⌘ + D` | 复制当前选中的节点 |
| `Delete` / `Backspace` | 删除当前选中的节点 |

**右键菜单（在节点上右键）：**

| 选项 | 说明 |
|------|------|
| 复制节点 | 在当前位置旁创建副本 |
| 删除节点 | 删除节点及相关连线 |
| 设为起点 | 将此节点设为流程起始节点 |
| 断开所有连线 | 移除该节点的所有进出连线 |

**画布操作：**

| 操作 | 说明 |
|------|------|
| 鼠标滚轮 | 缩放画布 |
| 拖拽空白区域 | 平移画布 |
| 框选（拖拽空白） | 选中区域内的多个节点 |
| 左下角控件 | 缩放控制、适应屏幕 |

### 8.8 导出与导入

**导出 JSON：**
1. 点击顶部工具栏的「更多」按钮
2. 选择「导出 JSON」
3. 浏览器下载包含完整流程配置的 JSON 文件

**导入 JSON：**
1. 点击「更多」→「导入 JSON」
2. 选择之前导出的 JSON 文件
3. 系统自动加载流程配置并在画布中展示

> 导出/导入可用于流程备份、跨环境迁移和团队协作。

### 8.9 保存流程

编辑完成后，点击右上角「保存」按钮，流程配置将保存到技能的 `workflow_config` 字段中。

### 8.10 设计示例

**简单查询流程：**
```
[工具调用: 查询产线] → [文本回复: 展示结果]
```

**带条件判断的流程：**
```
[工具调用: 查询状态] → [条件判断: 是否在产]
                           ├─ 是 → [文本回复: 在产信息]
                           └─ 否 → [文本回复: 已完成信息]
```

**复杂流程（带确认和并行）：**
```
[工具调用: 查询订单] → [用户确认: 确认操作]
                           ├─ 确认 → [并行执行]
                           │            ├─ [工具调用: 更新状态]
                           │            └─ [工具调用: 发送通知]
                           │         → [AI 生成: 生成总结]
                           │         → [文本回复: 操作完成]
                           └─ 拒绝 → [文本回复: 操作已取消]
```

## 九、对话测试

**路径：** 左侧菜单 → 对话测试

### 9.1 页面布局

- **左侧**：会话列表，展示所有历史对话

- 每个会话独立维护上下文
- 同一会话内支持多轮对话
- 点击左侧列表的历史会话可查看完整消息记录
- 可删除不需要的历史会话

## 十一、归一化规则

**路径：** 左侧菜单 → 归一化规则

归一化规则用于将用户输入中的自然语言表达转换为系统可理解的标准格式。

### 11.1 规则类型

| 类型 | 说明 | 示例 |
|------|------|------|
| 日期短语 | 自然语言日期 → 标准日期 | "上个月" → 2026-02-01~2026-02-28 |
| 状态映射 | 口语化状态 → 枚举值 | "在做了" → "进行中" |
| 参数转换 | 用户表达 → API 参数格式 | 产线编号格式化 |

### 11.2 管理操作

1. **新建规则**：选择规则类型，配置匹配模式和转换规则
2. **编辑规则**：修改已有规则的匹配条件或转换结果
3. **删除规则**：移除不再需要的规则
4. **初始化**：点击「初始化」加载系统内置的默认规则集

### 11.3 配置说明

每条规则包含：
- **规则编码**：唯一标识
- **分类**：日期短语 / 状态映射 / 参数转换
- **匹配模式**：正则或关键词
- **配置**：JSON 格式的转换规则

## 十二、常见问题

### Q: 登录后显示空白的仪表盘？

A: 需要先执行演示数据脚本 `python3 scripts/seed_demo.py` 创建初始数据。

### Q: 对话测试时提示"无法理解"？

A: 检查以下事项：
1. 是否已创建并发布技能
2. 技能的匹配关键词是否包含用户输入中的关键词
3. 技能关联的工具链是否正确配置

### Q: 连接器测试失败？

A: 排查步骤：
1. 检查基础 URL 是否正确
2. 确认认证信息是否配置正确
3. 如果外部 API 不可用，可开启 Mock 模式测试

### Q: 大模型测试无响应？

A: 排查步骤：
1. 确认 API 地址可访问
2. 确认 API Key 有效且未过期
3. 如使用 LM Studio，确认本地模型服务已启动

### Q: 流程编排保存后没有生效？

A: 确保：
1. 流程中设置了起点节点（第一个添加的节点自动为起点，也可右键设为起点）
2. 节点之间正确连线
3. 技能已发布
4. 保存按钮已点击（右上角）

### Q: 如何备份流程配置？

A: 在流程编排页面，点击工具栏「更多」→「导出 JSON」，保存 JSON 文件即可。恢复时使用「导入 JSON」。

### Q: 数据库如何迁移？

A: 使用 Alembic 数据库迁移工具：
```bash
cd backend
alembic revision --autogenerate -m "描述修改内容"
alembic upgrade head
```
