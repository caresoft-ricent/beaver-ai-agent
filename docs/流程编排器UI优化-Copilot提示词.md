# 流程编排器 UI/UX 优化需求

## 技术约束（必须遵守）

本项目基于 **@xyflow/react（React Flow v12+）** 开发。所有改造必须通过 React Flow 官方扩展机制实现，确保后续版本升级无痛：

- 所有节点视觉改造必须通过 **Custom Node 组件 + nodeTypes 注册**实现，不要修改默认节点
- 所有连线改造通过 **Custom Edge 组件 + edgeTypes 注册**实现
- 连接端口通过 **`<Handle>`** 组件自定义样式和位置
- 画布背景使用官方 **`<Background />`** 组件（`variant="dots"`），仅做样式覆盖
- 缩放控件使用官方 **`<Controls />`** 组件，仅做样式覆盖
- 缩略图使用官方 **`<MiniMap />`** 组件，仅做样式覆盖
- 框选使用内置 **`selectionOnDrag`** 属性，不要自己实现框选逻辑
- 拖拽添加节点使用 **`reactFlowInstance.screenToFlowPosition()`** 做坐标转换
- 状态管理只使用官方文档公开的 hooks：`useReactFlow()`、`useNodes()`、`useEdges()`、`useOnSelectionChange()` 等
- **禁止**：fork 或 patch React Flow 源码、依赖未文档化的内部 CSS class 或内部 store 字段、直接操作 `useStoreApi()` 的 internal 字段

---

## 背景
这是一个意图/技能流程编排功能，用于可视化编排 AI 工作流。当前已实现基础功能（左侧节点面板、中间画布拖拽连线、右侧属性配置面板），需要从用户体验和视觉专业度两方面进行优化，使其达到商业级 SaaS 产品水准。

## 一、节点视觉体系优化

### 1.1 节点卡片重设计
当前节点只有边框颜色区分，内容层次不够。请优化为：

```
每个节点卡片结构：
┌─ 顶部色带（4px，节点类型专属颜色）──────────┐
│  [图标]  节点名称              [状态指示灯]   │
│  ─────────────────────────────────────────── │
│  简要描述文字（灰色小字）                      │
│  ─────────────────────────────────────────── │
│  输入端口 ●                        ● 输出端口 │
└──────────────────────────────────────────────┘
```

要求：
- 卡片背景统一白色，用顶部 4px 色带区分类型（不要用整个边框上色，太花哨）
- 圆角 12px，阴影 `0 2px 8px rgba(0,0,0,0.08)`，hover 时加深为 `0 4px 16px rgba(0,0,0,0.12)` 并微微上移 translateY(-1px)
- 选中状态：色带颜色对应的浅色描边 + 浅色背景
- 节点宽度统一 220px，高度根据内容自适应但最小 80px
- 连接端口（Port）：左侧为输入（实心圆 8px），右侧为输出（空心圆 8px），hover 时放大到 12px 并高亮
- "起点"标记改为节点左上角的小 badge，不要放在节点名称旁边

### 1.2 节点类型配色方案
使用统一的配色系统，每种类型有主色和浅色（用于选中背景）：

| 节点类型 | 图标 | 主色 | 浅色背景 | 说明 |
|---------|------|------|---------|------|
| 工具调用 | ⚡ 或 Zap 图标 | #2563EB (蓝) | #EFF6FF | 调用外部工具/API |
| 条件判断 | GitBranch 图标 | #F59E0B (琥珀) | #FFFBEB | 条件分支路由 |
| 并行执行 | GitMerge 图标 | #7C3AED (紫) | #F5F3FF | 并行执行多个分支 |
| 用户确认 | CheckCircle 图标 | #10B981 (绿) | #ECFDF5 | 暂停等待用户确认 |
| AI 生成 | Sparkles 图标 | #EC4899 (粉) | #FDF2F8 | 大模型推理生成 |
| 文本回复 | MessageSquare 图标 | #06B6D4 (青) | #ECFEFF | 模板文本直接输出 |

### 1.3 连接线优化
- 当前直线/折线连接改为 **贝塞尔曲线**（cubic bezier），视觉更流畅
- 线条颜色：默认 `#CBD5E1`（浅灰），hover 高亮为起始节点的主色
- 线宽：默认 2px，hover 3px
- 连接线上标注条件文本（如条件判断的分支条件），用小的 pill badge 显示在线的中点
- 拖拽创建连线时，显示虚线动画引导
- 连线端点用小三角箭头（▶）指示方向

## 二、左侧节点面板优化

### 2.1 布局改进
当前是平铺列表，改为更紧凑和专业的设计：

```
添加节点                    [收起 ←]
─────────────────────────────────
🔍 搜索节点...
─────────────────────────────────
▾ 流程控制
  ├ ⚡ 工具调用      拖拽添加
  ├ 🔀 条件判断      拖拽添加
  └ ⚡ 并行执行      拖拽添加

▾ 交互与输出
  ├ ✅ 用户确认      拖拽添加
  ├ ✨ AI 生成       拖拽添加
  └ 💬 文本回复      拖拽添加
─────────────────────────────────
ⓘ 拖拽节点到画布添加
  点击节点端口连线
```

要求：
- 面板宽度 240px，可折叠收起（只显示图标）
- 节点分组：「流程控制」和「交互与输出」两组，可折叠
- 每个节点项左侧显示对应颜色的竖线（3px）+ 图标 + 名称
- 节点项支持拖拽（drag），拖拽时显示节点的半透明预览
- 底部常驻操作提示文字
- 加搜索框（在节点多的时候有用）

### 2.2 拖拽交互
- 从左侧拖出节点时，画布上出现放置区域的高亮提示（网格点亮或半透明投影）
- 放下时有缩放弹跳动画（scale 0.9 → 1.05 → 1.0）
- 画布空白处右键也可以唤出「添加节点」的快捷菜单

## 三、右侧属性面板优化

### 3.1 面板头部
```
┌──────────────────────────────────┐
│ [色带颜色条]                      │
│ 🔀 条件判断            📌 ✕      │
│ node_2                           │
├──────────────────────────────────┤
```
- 顶部加与节点类型匹配的色带（和节点卡片一致）
- 显示节点 ID（小号灰色字），方便调试
- 📌 图标用于固定面板（固定后点击其他节点不会切换）
- ✕ 关闭面板

### 3.2 表单样式优化
- 所有表单 label 加 tooltip 说明（hover 显示详细解释）
- 输入框统一样式：高度 36px，圆角 8px，边框 `#E2E8F0`，聚焦时 `ring-2 ring-blue-200`
- 下拉选择器改为自定义组件：
  - 支持搜索过滤（节点列表可能很长）
  - 选中项显示节点图标+名称+类型标签，而不是只显示 "node_1" 这样的 ID
  - 示例：`⚡ 工具调用 (node_1)` 而不是 `node_1`
- 条件分支配置区域改为可视化的条件卡片：
  ```
  分支 1: 当 status = "success"  → [选择节点 ▾]
  分支 2: 当 status = "failed"   → [选择节点 ▾]
  + 添加分支
  ──────────
  默认分支（不匹配时）→ [选择节点 ▾]
  ```

### 3.3 不同节点类型的属性面板内容

**工具调用节点：**
- 工具选择（下拉，显示工具图标+名称+简介）
- 参数映射（key-value 表格，支持从上游节点输出中选择变量）
- 超时设置（秒）
- 失败重试次数

**条件判断节点：**
- 判断字段（支持从上游输出字段中选择，显示为面包屑路径如 `node_1 → result → status`）
- 条件分支列表（可增删、拖拽排序）
- 每个分支：条件运算符（= / != / contains / > / <）+ 值 + 目标节点
- 默认分支

**并行执行节点：**
- 并行分支列表（可增删）
- 每个分支选择要执行的节点/子流程
- 合并策略：等待全部完成 / 任一完成即继续

**用户确认节点：**
- 确认提示文本（富文本编辑器）
- 确认按钮文案（默认"确认"）
- 取消按钮文案（默认"取消"）
- 超时时间（可选）

**AI 生成节点：**
- 模型选择（下拉）
- Prompt 模板（多行文本框，支持 `{{变量}}` 插值高亮）
- Temperature / Max Tokens 滑块
- 输出格式（文本 / JSON / Markdown）

**文本回复节点：**
- 回复模板（多行文本框，支持 `{{变量}}` 插值高亮）
- 变量列表（显示可用的上游变量，点击插入）

## 四、画布交互优化

### 4.1 画布控件
- 左下角缩放控件改为竖排按钮组：`[+]` `[比例显示如 100%]` `[-]` `[适应画布 ⛶]`
- 右下角加 minimap（缩略导航图），显示所有节点的位置，当前视口用方框标注
- 支持快捷键：
  - `Space + 拖拽` = 画布平移
  - `Ctrl + 鼠标滚轮` = 缩放
  - `Delete` = 删除选中节点/连线
  - `Ctrl + Z` / `Ctrl + Shift + Z` = 撤销/重做
  - `Ctrl + D` = 复制选中节点
  - `Ctrl + A` = 全选

### 4.2 画布背景
- 将当前的纯白背景改为带点阵的网格背景（dot grid），点的颜色 `#E2E8F0`，间距 20px
- 节点对齐时自动吸附到网格，并显示红色对齐辅助线

### 4.3 多选与框选
- 支持鼠标框选多个节点
- 多选后可以整体拖动、删除、对齐（左对齐/顶对齐/等距分布）
- 多选时右侧面板显示批量操作选项

### 4.4 节点上下文菜单（右键）
```
复制节点          Ctrl+D
删除节点          Delete
──────────────
设为起点
断开所有连线
──────────────
查看运行日志
```

## 七、视觉风格总纲

- 整体色调：以中性灰白为基底（背景 #FAFBFC，面板 #FFFFFF），节点类型颜色作为点缀
- 字体：系统字体栈 `-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`，代码/变量用 `'SF Mono', 'Menlo', monospace`
- 间距系统：基于 4px 网格（4/8/12/16/20/24/32）
- 圆角：小组件 6px，卡片 12px，面板 0px（贴边）
- 阴影层级：面板用 `shadow-sm`，节点卡片用 `shadow-md`，弹出菜单用 `shadow-lg`
- 动画：所有 transition 统一 200ms ease，避免过度动画
- 不要使用彩色背景大面积铺色，保持克制、专业
