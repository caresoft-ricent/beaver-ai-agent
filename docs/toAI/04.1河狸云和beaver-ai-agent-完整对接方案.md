# 河狸云 × AI Kernel（beaver-ai-agent）完整对接方案（团队执行版 & 新会话承接版）

> 用途：
> 1. 给团队统一认知（Java / AI / 产品）
> 2. 给 Copilot / 新会话作为上下文输入
> 3. 避免重复解释核心架构

---

# 1. 一句话总纲

# **河狸云 = 业务宿主系统**
# **AI Kernel = 智能运行时内核**

不是谁替代谁，而是分层。

---

# 2. 当前核心问题（必须统一认知）

当前问题不在 AI 能力，而在：

- ❌ 上下文不标准（header + localStorage 拼）
- ❌ 没有 session runtime
- ❌ 没有 capability gateway
- ❌ 多租户模型未结构化
- ❌ AI 在“补系统缺口”

👉 导致：
- 配置工作量大
- 不可扩展
- 不可复用
- 不可平台化

---

# 3. 河狸云 Java 侧定位（最终形态）

河狸云未来只做 4 件事：

## 3.1 身份与上下文源
- identity
- memberships
- default_scope

## 3.2 能力提供方
- Capability Gateway
- 查询 / 写操作能力

## 3.3 元数据提供方
- 企业 / 项目 / 组织
- 模块 / 状态 / 字典

## 3.4 权限与数据域控制
- 谁能看
- 谁能做

---

# 4. AI Kernel 定位

AI Kernel 统一负责：

- Session Runtime
- Scope Runtime
- Capability Runtime
- Ontology / Mapping
- Rule / Scenario
- 对话 / 主动通知

# 5. 当前 Token / Profile 问题（关键）

## 5.1 当前模式

- JWT（身份）
- Header（enterprise / member）
- localStorage（当前企业）

👉 **不是一个正式对象**

---

## 5.2 核心问题

- ❌ 上下文不可信
- ❌ 不支持多 membership
- ❌ 无法跨租户
- ❌ 前端驱动

---

# 6. 目标模型（核心升级）

## 6.1 Ticket（Java 提供）

```json
{
  "user_id": "u1",
  "memberships": [...],
  "default_scope": {...}
}
```

👉 一次性给“全部权限范围”

---

## 6.2 Session（Kernel）

```bash
POST /session/exchange
```

生成：

- identity
- memberships
- active_scope

---

## 6.3 Active Scope（运行时）

```json
{
  "enterprise_id": "...",
  "member_id": "...",
  "module": "...",
  "project_id": "..."
}
```

👉 所有执行基于它

---

# 7. 租户切换机制（统一方案）

## 7.1 嵌入模式

```js
postMessage({ type: "scope_change" })
```

→ Kernel 调：

```bash
POST /session/activate-scope
```

---

## 7.2 独立模式（ai.ricent.com）

用户选择 → 同一个接口：

```bash
POST /session/activate-scope
```

---

## 核心原则

# **前端只触发，后端决定**

# 8. Gateway 设计（关键）


- scope

- data
- evidence
