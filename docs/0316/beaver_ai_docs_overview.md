## 本次修订的核心变化

### 变化一：Session Scope 基于真实数据
原版是抽象设计，修订版基于河狸云实际的 Header 字段、localStorage、rc_user / rc_enterprise 数据结构，每个字段都有明确来源和示例值。

### 变化二：第一阶段 Java 团队零改动
原版要求 Java 团队提供 3 类改造。修订版发现前端 Header 已携带完整上下文，AI 侧自行提取即可。

### 变化三：10 件事精简为 5 个 Stage
原版列了 10 件"最优先的事"并行推进。修订版严格串行，做完一个再下一个。

### 变化四：新增"每个 Stage 必须交付场景"纪律
防止架构升级脱离业务交付。

### 变化五：新增代码审计专题文档
基于 beaver-ai-agent 完整源码的深度审计，给出具体的技术改造建议。

## 总体判断

当前 `beaver-ai-agent` 已经具备完整的执行链路（流式引擎 + 工作流编排 + 证据链 + AG-UI 协议），超出了普通 POC 水平。

但它仍处在 **"Skill-first 的配置化 Agent 项目"** 阶段，还没有升级为 **"Ontology-first 的企业级 AI Kernel"**。

后续所有改造，都应围绕 5 个 Stage 严格串行展开：

```text
Stage 0: 地基固化（合并双引擎 + 补测试）
Stage 1: Scope 接入（BeaverSessionScope）
Stage 2: Capability 升级（Action → Capability）
Stage 3: 桥接落地 + 场景交付（Gateway + 客户场景）
Stage 4: 平台化（Skill 拆分 + Eval + 模板）
```

**每个 Stage 完成后才启动下一个。每个 Stage 必须同时交付至少一个可跑通的客户场景。**
